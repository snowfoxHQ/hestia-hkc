# HKC 路线图 (Roadmap)

记录 HKC 在 v1.0 之后计划中的方向。欢迎在 [Issues](../../issues) 里讨论、提建议或认领 ——
参与方式见 [CONTRIBUTING.md](CONTRIBUTING.md)。

> 标 ⭐ 的是优先级较高、也比较适合新贡献者上手的方向。

## v1.1 —— 近期（小步、稳）

**易用 / 部署**
- [ ] ⭐ **Docker 一键部署**：`Dockerfile` + `docker-compose.yml`，`docker compose up` 直接起后端 + 前端，免手动建 venv/装依赖。
- [ ] **语义搜索引导**：默认向量后端为 `stub`（无真实语义）；在界面更明确地提示，并提供切换到 `local`（真实语义向量）的引导。

**功能**
- [ ] **邻居展开接入界面**：`/search/neighbors` 后端已具备，把它接到 UI（v1.0 已接入「两点最短路径」）。
- [ ] **AI 综述流式输出**：改为边生成边显示（SSE），减少等待感。

**工程 / 社区**
- [ ] ⭐ **CodeQL 代码扫描** + **Dependabot** 依赖更新。
- [ ] **前端 E2E 测试进 CI**：把 Playwright 冒烟测试纳入 GitHub Actions。
- [ ] **验收脚本模型无关化**：`scripts` 里的真实-LLM 验收目前偏 Anthropic，改为任意 provider。

## v2 —— 中长期（较大 / 需单独设计）

**知识质量**
- [ ] **语义去重「第 2 档」**：对「同义不同形」的近重复做向量级归并（高阈值 + 保守策略，宁漏勿误）。
- [ ] ⭐ **ACE 覆盖度改向量匹配**：能力覆盖度目前靠概念关键词匹配，改为向量相似度，提升中文同义词的召回。
- [ ] **领域（domain）规整**：对 LLM 抽取出的细碎领域做归并 / 别名映射。

**生态**
- [ ] ⭐ **界面国际化 (i18n)**：中 / 英双语。
- [ ] **能力自定义界面**：目前编辑 `skill_taxonomy.json`，做成可视化编辑。

**生产化（公网 / 多人部署场景）**
- [ ] **多 worker 支持**：把 BM25 / 向量索引外置，支持多进程。
- [ ] **摄入任务持久化**：任务状态落库，重启不丢。
- [ ] **速率限制 / 成本护栏**：LLM 调用配额与每日上限。

**视觉**
- [ ] **分层多壳星球**：节点极多时按层渲染（`PLANET_LAYOUT` 已预留扩展点）。

## 刻意不做 (Non-Goals)

保持项目聚焦，这些方向**不在计划内**：

- ❌ **力导向布局** —— 会把球面拍扁成平面网，失去 3D 星球的灵动感。
- ❌ **外部向量库（Milvus / Pinecone 等）** —— 违背「本地优先、离线可用、零网络依赖」原则；坚持用本地 faiss。
- ❌ **低阈值激进自动合并去重** —— 会误合不同知识、污染知识库；知识的合并只在充分证据下由 KEE 决定。

## 如何贡献

欢迎从 v1.1 里标 ⭐ 的方向入手。提交前请阅读 [CONTRIBUTING.md](CONTRIBUTING.md)，并遵守两条硬性原则：

- **Principle 07**：知识的身份 / 合并 / 演化只能由 KEE 根据证据决定。
- **模型无关**：任何 LLM 都能接入，绝不硬编码某个具体模型。
